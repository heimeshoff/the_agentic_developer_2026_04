# Plan: Personal Finance & Budgeting App (Frontend Prototype)

## Context

Exercise One of "The Agentic Developer" workshop asks the team to build a personal finance app with no prescribed stack or features. The goal is breadth and exploration. This is a greenfield frontend-only prototype — no backend, no auth, data persisted in localStorage.

Target folder: `teams/laszlo-robert/exercise_one/`

---

## Stack

**Vite + React + TypeScript + Tailwind CSS + shadcn/ui**

- Fastest way to get a polished UI without design decisions
- No router needed — single-page tab navigation via `useState`
- shadcn/ui provides accessible, styled components out of the box

---

## Four Views (tab-switched in App.tsx)

| Tab | View | Purpose |
|---|---|---|
| Income | `IncomeView` | Add/list income sources |
| Budget | `BudgetView` | Category budgets + spend vs. budget |
| Savings | `SavingsView` | Goals with progress bars |
| Investments | `InvestmentsView` | Portfolio with gain/loss calc |

A `SummaryBar` (sticky footer or hero row) shows total income / total spend / net across all tabs.

---

## Data Model (localStorage key: `pf_data`)

```ts
type IncomeEntry     = { id, source, amount, frequency, date }
type BudgetCategory  = { id, name, budgeted }
type Transaction     = { id, categoryId, description, amount, date }
type SavingsGoal     = { id, name, targetAmount, currentAmount, deadline }
type Investment      = { id, name, assetType, shares, purchasePrice, currentPrice }

type PFData = {
  income: IncomeEntry[]
  budgetCategories: BudgetCategory[]
  transactions: Transaction[]
  savingsGoals: SavingsGoal[]
  investments: Investment[]
}
```

All IDs use `crypto.randomUUID()`.

## Bootstrap Commands

```bash
npm create vite@latest teams/laszlo-robert/exercise_one -- --template react-ts
cd teams/laszlo-robert/exercise_one
npm install
npm install -D tailwindcss @tailwindcss/vite
npm install class-variance-authority clsx tailwind-merge lucide-react
npx shadcn@latest init          # accept defaults, zinc theme, CSS vars: yes
npx shadcn@latest add button input label card progress badge dialog tabs table
npm run dev
```

## Implementation Order

1. Bootstrap + verify dev server shows Vite default
2. `types.ts` → `useLocalStorage.ts` (data layer)
3. `useFinanceData.ts` with seed data
4. `App.tsx` + `NavBar` with four placeholder views (tab switching works)
5. `IncomeView` end-to-end
6. `BudgetView` (spend-vs-budget math)
7. `SavingsView` (progress bars)
8. `InvestmentsView` (table + inline price edit)
9. `SummaryBar` (aggregates from all domains)
